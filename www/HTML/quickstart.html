ProVision: Quick start guide

Requirements

ProVision is designed for statistical analysis of label free proteomics data that was generated using the popular maxQuant computational proteomics platform. The only requirement to use ProVision is a file called protienGroups.txt. If your samples were not searched by maxQuant and you would like to use this platform, you can find how to do the searches yourself here.

Load data

If no data processing has been done a new file can be uploaded. Else refresh the page to wipe all data and re upload your file. As of this version, only MaxQuant output is supported. If you don't have your own data, we have provided tutorial data: proteinGroupsTutorial.txt

MaxQuant folder structure is as follows:

specified output folder > combined > txt > proteinGroups.txt

Initial processing

Pre processing of output data is important to the downstream statistical tests and accurate visualization but may also be the most unintuitive step of the pipeline. The default parameters are pre chosen to yield optimal results and can simply be followed as is. Options are however in place to alter the pre chosen values if that is desired. To follow the pipeline, start at the upload tab and follow the workflow till the end. The only step that user input is required would be the annotation step. A table will be rendered where replicates need to be annotated with the same name and there is also a column for axis labels. It is important that axis labels are unique, i.e. rep1, rep2 rep3 etc. The only optional part is imputation, we do however recommend that this is done as it provides for better statistical testing. Imputation will assign values to proteins that have not been detected based on a normal distribution. If a mistake was made the page can be reloaded to start again. Lastly, the data is reactive and changes will propagate throughout each step. This allows for exploratory data analysis and experimentation of different parameters in ProVision if required.

Generating graphs

The strength of R is the graphing controls as well as the statistics. We have implemented the majority of these controls in the user interface. Based on the type of plots generated, options for graph control may be in another menu item or in a dropdown menu above the plots. The choices for the graphs are persistent and will be retained across multiple graphs of the same type.

Statistical testing

ProVision uses the linear models for micro array data (LIMMA) for statistical tests. This is a powerful framework designed specifically for omics data and offers multiple advantages over normal ANOVA or t-tests. Again the default parameters represent options that are typically used with options to shift. These options can be changed after calculation as well. Corrected p-values, or q-values are used for all the significant calls and by default is calculated with Benjamini-Hochberg false discovery rate.

Exporting

Tables generated in ProVision can be exported by either choosing the export options in the plot or by navigating to the download button. Excel output is the default, however options exist for text file exporting as well with various separator choices. Plots can be exported to PNG by right clicking on the plot its self. However, more control over plot exports are available in the download menu such as file types and resolution. Low resolution = 72dpi, middle = 300 dpi and high = 320 dpi.
